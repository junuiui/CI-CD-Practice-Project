# CI/CD 기초

## CI (Continuous Integration) - 지속적 통합

> 개발자들이 각자 만든 코드를 하나의 저장소 (GitHub, GitLab, ...)에 합칠 때, 그 코드가 멀쩡한지 자동으로 확인하는 과정

- *Build*
  - 작정한 코드를 실행 가능한 파일로 변환 (Compile, Build, ...)
- *Test*
  - 새로 짠 코드가 기존 기능을 망가뜨리지 않는지, 에러는 없는지 자동으로 검사
- *목적*
  - 코드 충돌을 빨리 찾고, 항상 "배포가 가능한 상태"의 코드를 유지하는 것

---

## CD (Continuous Delivery/Deployment) - 지속적 제공/배포

> CI를 통과한 깨끗한 코드를 실제 사용자에게 전달하는 과정

- *Continuous Delivery (지속적 제공)*: 
  - 배포 준비는 끝났지만, 실제 배포는 사람이 '승인' 버튼을 눌러야 함.
- *Continuous Deployment (지속적 배포)*: 
  - 사람이 개입하지 않고, 모든 테스트를 통과하면 바로 서버에 반영됨.
- *목적*: 
  - 개발자가 서버에 접속해서 수동으로 파일을 올리는 삽질(Manual work)을 없애는 것.

---

## CI/CD 전체 흐름 (Pipeline)

> 이 모든 과정이 하나의 파이프라인(Pipeline)처럼 연결되어 흐름.

- *Code*: 
  - 개발자가 코드를 `git push` 함.
- *Build*: 
  - CI 서버(GitHub Actions 등)가 코드를 가져와서 빌드함.
- *Test*: 
  - 자동화된 테스트 코드가 실행됨. (에러 나면 여기서 멈춤)
- *Release/Deploy*: 
  - 테스트 통과 시 Docker 이미지를 만들거나 서버(AWS EC2/S3 등)에 파일을 전송함.
